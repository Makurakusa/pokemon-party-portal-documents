```
tests/

    auth/

    setup-teardown/           ← 各テストの setup/teardown 定義

    helpers/
    constants/
    types/
```

### ログイン画面
Auth.js（NextAuth.js）を用いて認証
- Google / GitHub 等のソーシャルログイン対応
- JWT セッションでバックエンド（Hono）と連携

### トップページ（見た目のみ）
- 外部リンク一覧のレイアウトのみ実装（データなし）
- ログイン後のリダイレクト先として機能を確認する用途

### エンドポイント

| メソッド | パス | 認証 | 概要 |
|---|---|---|---|
| POST | /auth/register | 不要 | ユーザー登録 |
| POST | /auth/login | 不要 | ログイン（JWT 発行） |
| GET | /auth/me | 必要 | 認証済みユーザー情報取得 |
| POST | /auth/logout | 必要 | ログアウト |

## CI/CD

GitHub Actions を使用。フロントエンド・バックエンドそれぞれのリポジトリに設置。

### パイプライン構成（共通）

```
PR 作成時
  └── lint ──┐
  └── typecheck ──┼── （並列）
  └── test ───┘

main マージ時
  └── lint → typecheck → test → deploy:staging（自動）

tag push 時（v*.*.*）
  └── lint → typecheck → test → deploy:production（手動承認）
```

### フロントエンド（pokemon-party-portal-frontend）

| ステップ | 内容 |
|---|---|
| lint | ESLint |
| typecheck | `tsc --noEmit` |
| test | Vitest |
| deploy:staging | Vercel Preview（PR ごとに自動生成） |
| deploy:production | Vercel Production（Vercel GitHub 連携 / tag push） |

### バックエンド（pokemon-party-portal-backend）

| ステップ | 内容 |
|---|---|
| lint | ESLint |
| typecheck | `tsc --noEmit` |
| test | Vitest |
| deploy:staging | Railway Deploy Hook（staging 環境） |
| deploy:production | Railway Deploy Hook（本番環境 / 手動承認） |
